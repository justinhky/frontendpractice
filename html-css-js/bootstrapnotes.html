Personal notes in learning bootstrap

the mindset in the classcification of classes on bootstrap-personal experience

The main concept of Bootstrap is to categrize different HTML elements, based on their utilities. Then, they will continue to categrize each HTML element based on their features.

  1. base classes or object classes

    2. Bootstrap uses the base classes to define the styles of HTML elements in a direct way. Like .table, .grid, .col, .row, etc. They have the common features of visible in HTML

  2. opt-in classes

    3. the opt-in classes are optional modifier classes or custom styles. Because many styles are not inherited in Bootstrap, they utilize the opt-in classes to custom nested elements, such as nested tables. So, they can be independent from the parent.

  3. contextual classes

    4. Contextual classes are used to tyle "items inside the elements", such as cells inside the table. Usually they style affects like color, fonts, etc.

    The format is usually "base class-categories", such as table-primary, table-Secondary, table-success, table-warning, etc

    By the way, Bootstrap categorizes colors in primary, Secondary, success, danger, warning, info, light, dark.

  4. variants

    5. Bootstrap defined a list of variants for different HTML elements. Some of them are global, while others are used for specified classes. For example, table-striped is usded for striped rows.

customize-sass

importing

In your custom.scss, you’ll import Bootstrap’s source Sass files. You have two options: include all of Bootstrap, or pick the parts you need. The author encourage the latter.

option A

      
        // Custom.scss
        // Option A: Include all of Bootstrap

        // Include any default variable overrides here (though functions won't be available)

        @import "../node_modules/bootstrap/scss/bootstrap";

        // Then add additional custom code here

option B


      // Custom.scss
      // Option B: Include parts of Bootstrap

      // 1. Include functions first (so you can manipulate colors, SVGs, calc, etc)
      @import "../node_modules/bootstrap/scss/functions";

      // 2. Include any default variable overrides here

      // 3. Include remainder of required Bootstrap stylesheets
      @import "../node_modules/bootstrap/scss/variables";
      @import "../node_modules/bootstrap/scss/mixins";
      @import "../node_modules/bootstrap/scss/root";

      // 4. Include any optional Bootstrap CSS as needed
      @import "../node_modules/bootstrap/scss/utilities";
      @import "../node_modules/bootstrap/scss/reboot";
      @import "../node_modules/bootstrap/scss/type";
      @import "../node_modules/bootstrap/scss/images";
      @import "../node_modules/bootstrap/scss/containers";
      @import "../node_modules/bootstrap/scss/grid";
      @import "../node_modules/bootstrap/scss/helpers";

      // 5. Optionally include utilities API last to generate classes based on the Sass map in `_utilities.scss`
      @import "../node_modules/bootstrap/scss/utilities/api";

      // 6. Add additional custom code here

Containers

types of containers in bootstrap

  • .container: it's the default container, which sets a max-width at each responsive breakpoint
  • .container-fluid: it's the fluid container, which is 100% width at all breakpoint
  • .container-{breakpoint}: it's the responsive container, which is width: 100% until the specified breakpoint
